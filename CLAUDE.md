# CLAUDE.md

这个文件每次都会被 Claude Code 自动读到,是本仓库的工作上下文。下面的规则是硬约束,默认情况下都要遵守。

## 仓库定位

- 这是 Claude Code routine 的**内容生产工作区**,不是代码仓库,不要往里写代码或配置脚手架。
- 所有产出最终都要交给人 review,**不会自动发布**到任何外部平台。
- 用户会人工从这里挑选有价值的产出,搬到真正的网站仓库去发布。

## 文件命名规范

- `keyword-research/` 下:`YYYY-MM-DD.md`(例 `2026-05-17.md`),一天一份,不要塞别的文件名。
- `articles/drafts/` 下:`YYYY-MM-DD-slug.md`,slug 用英文小写连字符(例 `2026-05-17-genshin-5-7-leak`)。

## 分支规范

- **只允许 push 到 `claude/` 前缀的分支**,例如 `claude/keywords-2026-05-17`、`claude/draft-genshin-5-7-leak`。
- **禁止直接在 `main` 分支提交**任何内容。如果当前在 `main`,先切到 `claude/...` 分支再开始写。
- 一次产出对应一个 PR,标题简要说明产出内容。

## 写作风格

- **中文为主**,面向中文游戏玩家社区。
- 口语化但信息密度要高。避免"非常重要"、"值得关注"、"备受瞩目"这种没有信息量的空话。
- 关键词分析必须**给出可验证的数据来源**(具体平台、具体榜单、具体时间点),不能凭空判断"很火"、"热度高"。如果数据源拿不到,在调研日志里如实记录,不要编。

## 禁止事项

- 不要修改 `README.md` 和 `CLAUDE.md`,除非用户明确要求。
- 不要删除其他 routine 创建的文件,即使看起来过期了也只能挪到 `archive/`。
- 不要直接往 `main` 提交任何内容。

## 提交信息规范

格式:`类型: 简要说明`(中文),类型只能是以下几种:

- `keywords:` —— 关键词调研产出,例 `keywords: 2026-05-17 每日调研`
- `draft:` —— 文章草稿,例 `draft: 原神 5.7 前瞻直播看点`
- `chore:` —— 仓库维护(目录调整、模板更新等)

## 模板使用

写关键词调研报告时,以 `templates/keyword-research-template.md` 为骨架,按里面的 H2 区块填内容,不要随意增删一级结构。模板里的 `<!-- ... -->` 注释是填写指引,产出时删掉。
